# Тест-спецификация: проведение документа

## Область проверки
Документ задает минимальный набор проверок для сценария 'проведение документа'.

## Обязательные проверки
1. Корректный запрос успешно обрабатывается сервером.
2. Невалидный запрос возвращает понятный error code.
3. Повторный вызов не ломает состояние, если по контракту ожидается идемпотентность.
4. Состояние read-model после операции соответствует ожидаемому бизнес-результату.
