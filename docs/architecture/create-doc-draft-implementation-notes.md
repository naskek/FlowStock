# Заметки по реализации: создание черновика документа

## Цель
Документ описывает рабочие заметки по реализации сценария 'создание черновика документа'.

## Базовые принципы
- минимальные диффы вместо большого рефакторинга
- сохранение product invariant по `ledger` и `Close`
- перенос бизнес-логики в canonical server path
- сохранение idempotency и понятного audit trail

## Что проверять после изменений
- build для `FlowStock.Server` и `FlowStock.App`
- релевантные server tests
- отсутствие регрессии в WPF, TSD и PC сценариях
