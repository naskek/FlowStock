# Текущее состояние: создание черновика документа

## Назначение
Этот документ фиксирует текущее состояние сценария 'создание черновика документа' в репозитории FlowStock.

## Что уже сделано
- Основной non-KM runtime в WPF переведен на server API-first поток.
- Инварианты `ledger` и `Close` остаются обязательными.
- Для пользовательских сценариев приоритет отдается canonical endpoint-ам сервера.

## Что еще учитывать
- Исторически часть поведения могла идти через legacy helper-path или compatibility-fallback.
- При изменениях нужно проверять согласованность WPF, TSD и PC web там, где сценарий общий.
- При расхождении приоритет имеют `docs/spec.md`, `docs/spec_orders.md` и код.
