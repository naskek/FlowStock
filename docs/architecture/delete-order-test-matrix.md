# Матрица тестов: удаление заказа

## Базовые группы сценариев
- позитивный сценарий
- валидация входных данных
- повторный вызов и идемпотентность, если она требуется
- совместимость клиента с ожидаемым server contract

## Что обязательно покрывать
- переходы статуса объекта
- влияние на `ledger`, если сценарий меняет складское состояние
- поведение на уже закрытых или уже обработанных сущностях
- API metadata, если сценарий использует `api_docs` или `api_events`
