# Миграция WPF: обновление заказа

## Цель
Документ фиксирует состояние миграции WPF-сценария 'обновление заказа' на server API-first модель.

## Что считается целевым состоянием
- WPF использует server API как основной путь
- прямая бизнес-логика через PostgreSQL из окна или экрана отсутствует
- legacy fallback либо удален, либо оставлен только как явно временный compatibility-слой

## Критерий завершения
Сценарий считается завершенным, когда работа пользователя не требует прямого business-path в БД и вся операция проходит через серверный контракт.
